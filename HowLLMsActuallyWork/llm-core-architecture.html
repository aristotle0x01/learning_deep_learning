Decoder-only LLM 核心架构

基础逻辑视图：每个 Attention Head 独立执行自己的 Q/K/V 参数投影、RoPE 与 Attention；各 Head 输出最终由 WO 汇总。

Attention Head 的计算边界

  • • 每个 head 使用自己的 WQ_h / WK_h / WV_h 参数切片
  • • RoPE 分别应用于该 head 的 Q_h 与 K_h，V_h 不旋转
  • • 每个 head 独立计算 attention 权重与加权 V_h
  • • 所有 head 输出最后 Concat，并由 W_O 混合回 d_model

Residual Stream 是主干

  • • Token embedding 初始化每个位置的 hidden state
  • • Attention 与 FFN 都读取当前状态并产生增量更新
  • • LayerNorm / RMSNorm 提供规范化读取视图，本身不写入主干
  • • 后续层读取的是此前所有更新的累积结果

常见优化，不进入基础主流程

  • • KV Cache：自回归生成时复用历史 token 的 K/V
  • • GQA / MQA：多个 Query heads 共享较少的 KV heads
  • • MoE：用路由选择少数 FFN experts 替换 Dense FFN
  • • Fused kernels：合并投影与算子执行，但不改变逻辑含义
