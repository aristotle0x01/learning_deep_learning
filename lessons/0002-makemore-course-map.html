Makemore 学习地图

目标不是先钻细节，而是知道每一讲在整个神经网络学习路径中承担什么角色。

一句话：makemore 是 Karpathy 用“字符级生成名字”这件小事，逐步搭出语言模型训练的主线：从 bigram，到 MLP，到训练稳定性，到手写反传，再到 WaveNet 风格的深层结构。它前面接 micrograd，后面接 GPT。

总路线

micrograd先理解标量级自动求导、反向传播、loss 和参数更新。
makemore把这些概念搬到字符级语言模型和 PyTorch Tensor 上。
WaveNet-style从浅层 MLP 过渡到更深、更有层次的网络。
GPT在语言模型框架上引入 self-attention 和 Transformer。

1. 大体内容和解决问题

课程位置 大体内容 它解决什么问题 第一遍重点
Lecture 1: micrograd 从零实现一个很小的自动求导引擎，用 Value 表示计算图节点。 先让你知道 loss.backward() 背后的链式法则和梯度累加，不然 makemore 后面的训练循环会像黑盒。 理解 forward、loss、backward、zero grad、step 这条训练循环。
Lecture 2: makemore Part 1, Bigram 用一个字符预测下一个字符；先用计数表，再把同一件事改写成 Tensor 训练。 建立语言模型的基本框架：输入上下文、预测下一个 token、采样、负对数似然 loss。这里开始把“生成文本”变成“分类下一个字符”。 抓住 x -> logits -> probabilities -> loss，不用纠结模型很弱。
Lecture 3: makemore Part 2, MLP 把 bigram 升级成多字符上下文：context window、embedding、hidden layer、softmax。 解决 bigram 只能看一个字符的问题；模型开始学习字符组合的连续表示，而不是只靠计数表。 看懂 embedding 是“字符 id 到向量”的查表，train/dev/test 是为了调参不污染 test。
Lecture 4: makemore Part 3, Activations, Gradients, BatchNorm 观察深层 MLP 的激活值和梯度分布，调整初始化，并引入 BatchNorm。 解释为什么神经网络不是“堆层数就能训好”：数值尺度不对，激活会饱和，梯度会过小/过大，训练会不稳定。 重点是诊断思路：看激活、看梯度、看参数更新比例，而不是死记 BatchNorm 公式。
Lecture 5: makemore Part 4, Backprop Ninja 不用 PyTorch autograd，手动推 cross entropy、linear、tanh、BatchNorm、embedding 的 Tensor 级反传。 把 micrograd 的标量反传升级到真实网络里的批量矩阵反传。以后看到 autograd 结果不对，知道从哪里怀疑。 第一遍知道“每一步 forward 都有对应 backward”即可，具体公式可以二刷。
Lecture 6: makemore Part 5, WaveNet 把 MLP 做深，变成类似 WaveNet 的层次结构，并更正式地使用 torch.nn 组织模型。 解决简单 MLP 对上下文处理太扁平的问题：让模型用层次化方式融合字符上下文，同时训练流程更接近真实项目。 重点跟住 shape：batch、time/context、channel/embedding 这些维度如何流动。
Lecture 7: GPT 从零实现 GPT。 makemore 已经给了语言模型、训练循环、embedding、采样、张量 shape 的基础，GPT 主要新增 self-attention 和 Transformer block。 学习时不要把 GPT 看成完全新东西，它仍然是在做“根据上下文预测下一个 token”。

2. 联系和递进关系

micrograd -> makemore

micrograd 解决“梯度怎么来”；makemore 开始解决“把梯度用在真实一点的语言模型训练里”。前者是标量计算图，后者是 Tensor 计算图。

带走：训练不是魔法，就是 forward 得到 loss，backward 得到 grad，再更新参数。

bigram -> MLP

bigram 只看一个字符，像很短视的统计表；MLP 看多个字符，并把字符变成 embedding。模型从“数频率”进入“学习表示”。

带走：语言模型的核心任务没变，变的是上下文长度和参数化方式。

MLP -> BatchNorm

MLP 能训练后，下一步不是立刻换高级模型，而是先问：训练健康吗？激活是否饱和？梯度是否消失？更新是否过猛？

带走：深度学习工程里，诊断训练状态和模型结构本身一样重要。

BatchNorm -> 手写反传

BatchNorm 让 forward 复杂起来，也让 backward 更不直观。手写反传的目的不是日常替代 autograd，而是训练你能读懂梯度流。

带走：会用 PyTorch 是一层，知道它在链式法则上做什么是另一层。

MLP -> WaveNet-style CNN

普通 MLP 把多个字符 context 压平处理；WaveNet 风格结构用层次化方式组合邻近字符，再组合更大的局部片段。

带走：模型结构开始利用序列的局部层次，而不是只把 context 当一坨向量。

makemore -> GPT

GPT 的任务仍然是预测下一个 token。makemore 已经练过 token 化的雏形、embedding、logits、loss、采样、训练循环和 shape 管理。

带走：GPT 的新增核心是 self-attention/Transformer，不是重新发明语言模型任务。

建议学习顺序

第一遍：按顺序看，目标是能说出每讲解决了上一讲的什么短板。

第二遍：重点复现 notebook，尤其是 Part 2 的训练循环、Part 3 的诊断图、Part 5 的 shape 流动。

遇到卡点：优先问三个问题：输入 shape 是什么？输出 logits 表示什么？loss 对哪个参数产生梯度？

学习时抓住什么

第一遍不要追求每个张量维度都立刻记住。更重要的是抓住这条主线：

数据如何变成训练样本 -> 模型如何给出下一字符概率 -> loss 如何衡量预测差距 -> backward 如何给参数梯度 -> optimizer 如何更新参数 -> 如何采样生成新名字。
