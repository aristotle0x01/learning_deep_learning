MLE：从计数概率到神经网络 loss

这节只解决一个问题：为什么“让真实样本概率最大”会变成 NLL/cross-entropy loss。

一句话：Maximum Likelihood Estimation, MLE，就是选择参数，让已经观察到的训练数据在模型下尽可能高概率。

1. probability 和 likelihood 的区别

Probability

参数已经固定，问某个数据发生的概率是多少。

例：已知硬币正面概率 p=0.6，问 10 次里 7 次正面的概率。

Likelihood

数据已经观察到了，反过来问哪个参数最能解释这些数据。

例：已知 10 次里 7 次正面，问 p=0.5、p=0.7、p=0.9 哪个更合理。

2. 数学骨架

有一批样本：

D = {x_1, x_2, ..., x_n}

模型带参数 θ，给每个样本一个概率：

p_θ(x_i)

如果先假设样本独立，整批数据发生的概率就是乘起来：

L(θ) = Π_i p_θ(x_i)

MLE 选择让这个乘积最大的参数：

θ_hat = argmax_θ L(θ)

实际优化时通常取 log，把乘法变加法：

log L(θ) = Σ_i log p_θ(x_i)

神经网络里习惯最小化 loss，所以取负号：

NLL(θ) = -Σ_i log p_θ(x_i)

3. 和 makemore bigram 的关系

在 bigram 里，一个样本不是整个人名，而是一个字符转移：

当前字符 c1 -> 下一个字符 c2

counting 模型直接从训练集统计：

P(c2 | c1) = count(c1, c2) / count(c1)

这本身就是 categorical / multinomial 模型下的 MLE：频率最高的转移，就被赋予更大的概率。

计算 loss 时仍然要遍历每一次真实出现的 bigram：

NLL = -mean_t log P(next_t | current_t)

这就是你前面看到的关键点：重复出现的 bigram 不是重复计算，而是在最大化整份数据的 likelihood。

4. 为什么 neural net 也还是 MLE

后面 neural bigram / MLP 不再直接用 counts 得到概率，而是：

input -> logits -> softmax -> probs

但 loss 没变，仍然是让真实下一个字符概率变大：

loss = -log P_θ(true_next_char | context)

区别只是：

模型概率从哪里来如何估计参数
counting bigram计数归一化闭式解：count / row_sum
neural bigram / MLP神经网络 softmax 输出梯度下降：backward + update

5. 应用场景

场景MLE 在估计什么直觉
硬币/点击率/转化率Bernoulli 参数 p成功次数 / 总次数。
bigram / Markov 转移P(next | current)某行里各转移次数归一化。
正态分布均值、方差均值的 MLE 是样本均值。
线性回归回归权重若假设高斯噪声，最小二乘就是 MLE。
logistic / softmax 分类分类边界参数最小化 cross-entropy 等价于最大化真实标签概率。
语言模型token 预测参数最大化真实下一个 token 的概率。
Naive Bayes词在类别下的概率计数估计，但需要 smoothing 避免未见词概率为 0。
混合模型 / EM带隐变量的分布参数直接 MLE 难解时，用 EM 迭代提高 likelihood。

6. 常见坑

未见事件概率为 0：如果某个 bigram 没出现，MLE 会给 0 概率；一旦测试时遇到它，-log(0) 就爆掉。所以会用 Laplace smoothing。

模型太灵活会过拟合：MLE 只管训练数据 likelihood，不自动保证泛化。神经网络里需要 validation、regularization、early stopping。

假设很重要：MLE 的结果依赖你选的概率模型。假设高斯噪声会导出最小二乘；假设 Bernoulli 会导出 logistic loss。
